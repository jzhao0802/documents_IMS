In the following, all Formulary Focus macros and output datasets are documented.

 

FF_Analysis(Datadirname, Dataname, Directory)

 

Datadirname:      directory with input SAS datasets and output SAS analysis

Dataname:         input file name and extension for all output

Directory:        output directory explicitly set for purpose of exporting data into EXCEL etc. (set same as Datadirname, for convenience)

 

This is the main Macro. All other macro functions are called from this macro. This macro defines the raw input dataset, the extensions for all output files, and input and output directories for SAS datasets, and the output directory for EXCEL files. These are as defined above.

 

In order of occurrence in main macro, they are:

  1. mergemapfiles(indata, outdata)

calls the following macro twice:

mergedata(indata, outdata, mapping, sortbyvar)

indata:     input data set (&Datadirname.Dataname)

outdata:    output data set (&Datadirname.mappeddataname_&Dataname)

mapping:    mapping file to be merged with the input dataset

sortbyvar: variable by which input is sorted for merging

 

output 1: mappeddata_&Dataname

 

The macro mergemapfiles merges input dataset (original Formulary Focus data) with both the mapping files for Product Notes and Formulary Status.

The original Formulary Focus Dataset has many separate Product Notes with Descriptions that could be consolidated into one category. The mapping file for Product Notes does this consolidation and sets up new Product Notes with new Description. The original Formulary Focus Dataset does not have codes for Formulary Status. The mapping file for Formulary Status sets up these codes using formulary_status variable.

 

  1. transposedata(indata, outdata)

indata:     input data set (&Datadirname.mappeddataname_&Dataname)

outdata:    output data set (&Datadirname.TherapyFormStrength_&Dataname)

 

output 2: TherapyFormStrength_&Dataname

 

The macro transposedata creates dummies at LEVEL 1 (form_stren_level) for the dataset mappeddata_&Dataname and transposes the data to get the output dataset with one observation per classification level of:

Plan_number

Formulary_name

Therapeutic_area_code

Prod_code

Form_stren_level_code

 

We set:

-        missing tier = tier 10

-        dummy variable name for recoded prod_notes_code (code) = formcode

-        dummy variable name for recoded form_status_code = statuscode

-        dummy variable name for tier = tiercode

 

  1. multiple_prod_code(indata, outdata, myext, name)

indata:     input data set (&Datadirname.TherapyFormStrength_&Dataname)

outdata:    output data set

(&Datadirname.multiple_prod_codes_&Dataname)

      myext:      output directory for excel output file (&Directory)

      name:       name of excel file (multiple_prod_codes_&Dataname)

 

output 3:   SAS:        multiple_prod_codes_&Dataname

      EXCEL:      &Directory\multiple_prod_codes_&Dataname

     

The formulary focus input dataset is sorted by:

Plan_number

Formulary_name

Therapeutic_area_code

Prod_code

Formulary_stren_level_code

 

The macro multiple_prod_code checks for entries with multiple values of Prod_code (immaterial of Formulary_stren_level_code) when all prior categories are the same. Such entries are for different product codes within the same therapeutic area. All such entries are output to a SAS dataset and as an EXCEL spreadsheet.

 

  1. rollup1(indata, outdata)

indata:     input data set (&Datadirname.TherapyFormStrength_&Dataname)

outdata:    output data set (&Datadirname.Therapy_prod_&Dataname)

 

output 4: Therapy_prod_&Dataname

 

The macro rollup1 rolls up multiple values for Prod_code for the same Therapeutic_area_code (and all other categories also same) and produces an output data set with a single observation per classification. This observation carries with it the information for tier, Formulary status and Product code for all the multiple entries.

 

  1. multiple_therapy_areas(indata, outdata, myext, name)

indata:     input data set (&Datadirname.Therapy_prod&Dataname)

outdata:    output data set

(&Datadirname.multiple_therapy_areas_&Dataname)

      myext:      output directory for excel output file (&Directory)

   name:       name of excel file (multiple_therapy_Areas_&Dataname)

 

output 5:   SAS:        multiple_therapy_areas_&Dataname

      EXCEL:      &Directory\multiple_therapy_Areas_&Dataname

     

The formulary focus input dataset is sorted by:

Plan_number

Formulary_name

Prod_code

Therapeutic_area_code

 

Hence, note that we now sort first by Prod_code and then by Therapeutic_area_code. The macro multiple_therapy_areas checks for multiple therapy areas when all prior categories are the same. Such entries are for different therapeutic areas which have same Product codes for the prescriptions. All such entries are output to a SAS dataset and also as an EXCEL spreadsheet.

 

  1. rollup2(indata, outdata)

indata:     input data set (&Datadirname.Therapy_prod&Dataname)

outdata:    output data set (&Datadirname.Prod_&Dataname)

 

output 6: Prod_&Dataname

 

Rolls up the multiple values of Therapeutic_area_codes for the same Prod_code (and all other categories also same) and produces an output data set with one observation per “Plan Number – Formulary Name – Product Code” classification. This observation carries with it the information for tier, Formulary status and Product notes for all the multiple entries.
